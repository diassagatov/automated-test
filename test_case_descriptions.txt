SOFTWARE TEST AUTOMATION - TEST CASE DESCRIPTIONS
=================================================

Project: Demo Blaze Website Test Automation
Framework: Robot Framework + SeleniumLibrary
Browsers: Local Chrome (via Selenium Manager) or remote BrowserStack sessions

TEST CASE 1: WEBSITE LOADS SUCCESSFULLY
--------------------------------------
Objective: Ensure the landing page renders and core header elements are present.

Steps:
1. Open https://www.demoblaze.com
2. Verify page title equals `STORE`
3. Ensure `Log in`, `Sign up`, and `Cart` links are visible

Expected Result: Page loads with all header controls ready for interaction.

TEST CASE 2: NAVIGATION ELEMENTS PRESENT
----------------------------------------
Objective: Validate that every primary navigation item is displayed.

Steps:
1. Open the site
2. Assert presence of `Home`, `Contact`, `About us`, and `Cart` links

Expected Result: All navigation anchors rendered and interactable.

TEST CASE 3: PRODUCT CATEGORIES VISIBLE
---------------------------------------
Objective: Confirm that Phones, Laptops, and Monitors category filters appear.

Steps:
1. Open the site
2. Check visibility of category links for Phones, Laptops, Monitors

Expected Result: Each category tab is present on the homepage.

TEST CASE 4: DEMO BLAZE MODAL FUNCTIONALITY
-------------------------------------------
Objective: Verify the sign-up and login modals open/close and expose expected inputs.

Steps:
1. Open the site
2. Click `Sign up`, wait for modal, assert username/password fields + button
3. Close modal (ESC)
4. Click `Log in`, wait for modal, assert username/password fields + button
5. Close modal

Expected Result: Both modals display required fields and close properly.

EXECUTION INSTRUCTIONS
----------------------
1. Create/activate venv:
   ```
   python -m venv venv
   venv\Scripts\activate
   ```
2. Install deps: `pip install -r requirements.txt`
3. Run tests:
   - All: `robot -d results tests/`
   - Smoke only: `robot tests/test_basic_functionality.robot`
   - BrowserStack:
     ```
     set BROWSERSTACK_USERNAME=...
     set BROWSERSTACK_ACCESS_KEY=...
     robot --variable USE_BROWSERSTACK:True -d results tests/
     ```

REPORTING
---------
- Results generated under `results/` (`report.html`, `log.html`, `output.xml`)
- Screenshots/logs captured automatically on failures (if any)
